## Problem Statement

developer-open-book은 97권의 프로그래밍 학습서를 픽셀 아트 RPG 도서관에서 제공하는 웹앱이다. 현재는 정적인 타일맵 도서관에서 책장을 클릭하고 마크다운을 읽는 것이 전부다. 픽셀 아트 세계관이라는 독보적인 차별점이 있지만, 인터랙션이 "클릭 → 읽기"로 제한되어 있어 세계관의 잠재력을 10% 정도만 활용하고 있다.

또한 별도 레포(develop-study-documents)에 Java 48, MSA 46, LLM 32, MySQL 31, JS/TS 22, Python 16권 등 195권 이상의 학습 문서가 추가 대기 중이다. 현재 97권에서 최대 300권 이상으로 확장될 예정이므로, 콘텐츠 추가가 쉬운 구조가 필수.

목표: 픽셀 아트 도서관을 "살아있는 학습 세계"로 고도화. NPC 가이드, 퀘스트 시스템, 읽기 진행 추적, 업적, 애니메이션, 그리고 궁극적으로 AI 튜터 NPC까지. 동시에 대규모 콘텐츠 확장을 수용하는 구조 설계.

## What Makes This Cool

전 세계 어디에도 "픽셀 아트 RPG 도서관에서 NPC와 대화하며 한국어로 코딩을 배우는" 경험은 없다. Codecademy는 게이미피케이션이 있지만 세계관이 없고, freeCodeCamp의 Learn to Code RPG는 데스크톱 게임이지 웹 학습 플랫폼이 아니다.

핵심 통찰: 대부분의 게이미피케이션 학습 사이트는 콘텐츠 위에 게임 메카닉을 "덧붙인다". 이 프로젝트는 다르다 — 게임 세계가 인터페이스 자체다. NPC, 퀘스트, 업적이 도서관 세계 안에서 자연스럽게 발생해야 한다. "Achievement Unlocked!" 팝업이 아니라, 사서 NPC가 "비동기 프로그래밍 서가를 다 읽으셨네요. 2층 고급 서가가 열렸습니다"라고 말하는 것.

## Constraints

- **프레임워크**: React 19 + TypeScript + Vite 8 (변경 불가)
- **배포**: GitHub Pages 정적 사이트 (서버리스, 백엔드 없음)
- **상태 저장**: localStorage (기기간 동기화 포기)
- **콘텐츠**: 마크다운 기반. 현재 97권, 확장 시 300권+
- **콘텐츠 소스**: develop-study-documents 레포에서 점진적 이관
- **미학**: 픽셀 아트 RPG 탑다운 시점 유지
- **비용**: AI 튜터는 Phase 2 이후, API 비용 고려 필요

## Premises

1. 픽셀 아트 도서관 세계관이 학습 동기를 높인다
2. NPC + 퀘스트는 기존 타일맵 위에 레이어로 추가 가능하다
3. 읽기 진행 추적은 localStorage로 충분하다
4. 콘텐츠 3중 복제 구조(content/, public/content/, src/content/)는 정리해야 한다
5. AI 튜터 NPC는 Phase 2 이후 작업이다
6. 콘텐츠는 300권 이상으로 확장되므로, 수동 라우트/페이지 등록 방식은 지속 불가능하다

## Approaches Considered

### Approach A: "World First" — 세계관 우선
애니메이션 + NPC + 대화 시스템 먼저. 시각적 임팩트 최대화.
- Effort: L / Risk: Med
- 장점: 첫인상 극대화, NPC가 퀘스트 매개체
- 단점: 학습 경험 자체 개선 없음, 데이터 레이어 부재

### Approach B: "Progress First" — 학습 시스템 우선
읽기 진행, 배지, 스트릭, 콘텐츠 구조 정리 먼저.
- Effort: M / Risk: Low
- 장점: 실질적 학습 개선, 데이터 레이어가 기반
- 단점: 시각적 "와" 요소 부족

### Approach C: "Full Stack" — 4단계 점진 구축 (선택됨)
모든 요소를 4단계로 나눠 점진적 구축. 각 단계 독립 배포.

## Recommended Approach: Full Stack 4-Phase Build

### Phase 1: Foundation (콘텐츠 아키텍처 + 진행 추적)

**1-0. 콘텐츠 확장 아키텍처 (핵심)**

현재 97권에서 300권+으로 확장 시 기존 방식(개별 TSX 파일 + routes.ts 수동 등록)은 지속 불가능. 자동화된 콘텐츠 파이프라인 구축.

```typescript
// src/data/shelves.ts — 책장(카테고리) 레지스트리
interface Shelf {
  id: string;                    // 'dart', 'java', 'msa' 등
  name: string;                  // 표시 이름
  icon: string;                  // 픽셀 아트 아이콘 경로
  color: string;                 // 테마 색상
  room: 'main' | 'east' | 'west' | 'upper' | 'archive';  // 도서관 내 위치
  sortOrder: number;
  locked?: boolean;              // 해금 필요 여부
  unlockCondition?: string;      // 해금 조건 (퀘스트 ID 등)
}

// 새 카테고리 추가 = shelves.ts에 1개 엔트리 추가 + public/content/{id}/ 폴더에 MD 넣기
```

콘텐츠 추가 워크플로우:
1. `public/content/{shelf-id}/` 폴더에 마크다운 파일 배치
2. `src/data/shelves.ts`에 책장 등록 (없으면 추가)
3. `src/data/books.ts`에 책 엔트리 추가
4. **routes.ts는 수동 편집 불필요** — 동적 라우트 매칭으로 전환

```typescript
// src/routes.ts — 동적 라우트 (300권+ 대응)
// 기존: 97개 개별 라우트 정의
// 변경: books.ts의 BOOKS 배열에서 자동 생성
import { BOOKS } from './data/books';
import { lazy } from 'react';

const BookPage = lazy(() => import('./pages/BookPage'));

export const routes: RouteConfig[] = [
  { path: '/', title: 'Developer Open Book', component: lazy(() => import('./pages/index')), category: 'Home' },
  ...BOOKS.map(book => ({
    path: `/${book.slug}`,
    title: book.title,
    component: BookPage,
    category: book.category,
    shelf: book.category,
    slug: book.slug,
  })),
];
```

|----------|------|-----------|------|-------------|
| Dart | 23 | — | 배포 완료 | 메인홀 |
| Flutter | 31 | — | 배포 완료 | 메인홀 |
| React | 43 | — | 배포 완료 | 메인홀 |
| Java & Spring | 0 | 48 | develop-study-documents | 동관 |
| MSA | 0 | 46 | develop-study-documents | 동관 |
| LLM/AI | 0 | 32 | develop-study-documents | 서관 |
| MySQL | 0 | 31 | develop-study-documents | 서관 |
| JS/TS | 0 | 22 | develop-study-documents | 메인홀 |
| Python | 0 | 16 | develop-study-documents | 서관 |

**도서관 공간 확장:**
- 현재: 1개 메인홀 (Dart, Flutter, React)
- 확장: 멀티룸 구조
  - 메인홀: 기존 3개 + JS/TS
  - 동관 (East Wing): Java & Spring, MSA — 백엔드/아키텍처 영역
  - 서관 (West Wing): Python, MySQL, LLM/AI — 데이터/AI 영역
  - 2층 (Upper): 퀘스트 해금으로 접근하는 고급 영역
  - 아카이브: Troubleshooting, 기타 문서
- 각 방은 별도 타일맵 + 별도 NPC 배치
- LibraryRoom → 방 간 이동은 출입구 타일 클릭

**1-1. 콘텐츠 구조 단일화**

목표: `content/` (루트) 제거, `src/content/` 제거. `public/content/`를 유일한 콘텐츠 소스로 사용. BookPage.tsx가 런타임에 fetch.

- `src/pages/dart-*.tsx` 24개 개별 파일 제거
- Dart도 Flutter/React와 동일하게 BookPage 범용 컴포넌트 사용
- 라우트 정의 방식 통일 (slug 기반, books.ts에서 자동 생성)

**마이그레이션 안전장치:**
- 기존 URL(예: `/#/dart-step01-overview-and-setup`)이 깨지지 않도록 slug 유지
- localStorage의 기존 bookId 키가 변경될 경우, 첫 로드 시 마이그레이션 함수 실행
- `src/store/migrate.ts`: 이전 형식 키 감지 → 새 형식으로 변환

**1-2. 진행 추적 시스템**
```typescript
// src/store/progress.ts
interface ReadingProgress {
  bookId: string;
  lastReadAt: string;       // ISO timestamp
  scrollPosition: number;   // 0-1 비율
  completed: boolean;
  timeSpentMs: number;      // 누적 읽기 시간
}

interface UserProgress {
  version: number;           // 스키마 버전 (마이그레이션용)
  books: Record<string, ReadingProgress>;
  badges: Badge[];
  streak: { current: number; lastDate: string; best: number };
  totalBooksRead: number;
  totalTimeMs: number;
}
```
- localStorage에 저장, React Context로 앱 전역 접근
- BookReader/BookPage에서 스크롤 위치 자동 저장
- 완독 감지: `max(30초, 추정 읽기시간의 50%)` + 스크롤 90% 이상
  - 500단어 미만 짧은 문서: 스크롤 바닥 도달만으로 완독 처리
- `version` 필드로 향후 스키마 변경 시 자동 마이그레이션

**1-3. 번들 크기 최적화**
- react-syntax-highlighter 언어팩 dynamic import 적용
- 초기 로드 번들 목표: 500KB 이하 (현재 ~993KB)
- 각 언어 하이라이터는 해당 콘텐츠 접근 시 lazy load

**1-4. 배지 시스템**
- 첫 책 읽기, 카테고리 완독, 연속 읽기 스트릭, 총 읽기 시간 등
- 배지 아이콘은 픽셀 아트 스타일 SVG
- LibraryRoom에서 책장의 읽은 책에 시각적 표시 (빛나는 효과 등)

**1-5. 사이드바/헤더 업데이트**
- Layout.tsx에 진행률 표시 (전체 읽은 비율)
- 현재 스트릭 카운터
- 최근 읽은 책 빠른 접근

### Phase 2: NPC + 퀘스트 시스템

**2-1. NPC 엔티티 시스템**
```typescript
// src/data/npcs.ts
interface NPC {
  id: string;
  name: string;
  role: 'librarian' | 'guide' | 'scholar' | 'merchant';
  room: string;                    // 어느 방에 있는지
  position: { x: number; y: number };
  sprite: string;
  dialogueTree: DialogueNode[];
}

interface DialogueNode {
  id: string;
  text: string;
  speaker: string;
  options?: { text: string; next: string; condition?: string }[];
  action?: 'giveQuest' | 'completeQuest' | 'giveHint' | 'openShelf';
  actionPayload?: GiveQuestPayload | CompleteQuestPayload | OpenShelfPayload;
}

type GiveQuestPayload = { questId: string };
type CompleteQuestPayload = { questId: string; rewardIds: string[] };
type OpenShelfPayload = { shelfId: string };
```
- LibraryRoom에 NPC 렌더링 레이어 추가
- 클릭 시 대화 UI (픽셀 아트 대화창)
- 대화 내용은 진행 상태에 따라 동적 변경
- NPC `schedule` 필드는 Phase 3에서 추가 (애니메이션 루프 필요)

**2-2. NPC 캐릭터**
- **사서 NPC (메인)**: 도서관 입구. 첫 방문자 가이드, 추천 책, 진행 상황 코멘트
- **Dart 학자**: Dart 서가 근처. Dart 관련 팁, 퀘스트 제공
- **Flutter 장인**: Flutter 서가 근처. 위젯 관련 힌트
- **React 연구원**: React 서가 근처. Hook 패턴 조언
- 확장 시: Java 마에스트로 (동관), Python 현자 (서관), AI 연구원 (서관) 등
- PixelLab MCP로 캐릭터 스프라이트 생성 가능

**2-3. 퀘스트 시스템**
```typescript
// src/data/quests.ts
interface Quest {
  id: string;
  title: string;
  description: string;
  giver: string;           // NPC id
  type: 'read' | 'streak' | 'explore' | 'complete_category';
  requirements: QuestRequirement[];
  rewards: QuestReward[];
  prerequisite?: string;   // 선행 퀘스트 id
}

interface QuestRequirement {
  type: 'read_book' | 'read_count' | 'streak_days' | 'time_spent' | 'complete_shelf';
  target: string | number;
}

interface QuestReward {
  type: 'badge' | 'unlock_area' | 'title' | 'cosmetic';
  value: string;
}
```
- 퀘스트는 NPC 대화를 통해 수락/완료
- 읽기 진행 데이터(Phase 1)와 자동 연동
- 예시 퀘스트:
  - "첫 발걸음": 아무 책 1권 읽기 → 배지 획득
  - "Dart 마스터": Dart 23권 전부 완독 → Dart 학자 칭호 + 동관 해금
  - "연속 학습": 3일 연속 읽기 → 특별 NPC 대화 해금
  - "전설의 독서가": 전체 완독 → 숨겨진 서가 해금
  - 확장 퀘스트: "백엔드 입문": Java Step 1-10 완독 → MSA 서가 해금

**2-4. 잠금/해금 메카닉**
- 초기에는 메인홀 (기초) 서가만 접근 가능
- 퀘스트 완료에 따라 동관/서관/2층 해금
- 각 방의 타일맵에 잠긴 출입구 시각화 (어둡게 / 자물쇠)
- 2층 = 별도 LibraryRoom 컴포넌트, 라우트/상태 플래그로 게이트

### Phase 3: 애니메이션 + 사운드

**3-1. 타일맵 애니메이션**
- CSS 애니메이션 또는 requestAnimationFrame 기반
- 촛불/랜턴 깜빡임 (CSS `@keyframes` + opacity)
- 먼지 파티클 (canvas overlay 또는 CSS)

**3-2. NPC 애니메이션 + 시간대별 이동**
- 아이들 모션 (2-3프레임 루프)
- 대화 시 말풍선 팝업 애니메이션
- NPC `schedule` 필드 추가: 실시간 시계 기반 (24h, 예: `'09:00-17:00'`)
- LibraryRoom에서 현재 시각에 따라 NPC 위치 결정

**3-3. 앰비언트 사운드**
- Web Audio API 기반
- 도서관 앰비언트 (페이지 넘기는 소리, 시계, 나무 삐걱거림)
- 효과음: 책 열기, 퀘스트 완료, 배지 획득, NPC 대화 시작
- 볼륨 조절 + 음소거 토글 (ThemeToggle 옆)
- 사운드는 public/audio/에 저장 (mp3/ogg, 작은 파일)

**3-4. 시각 효과**
- 읽기 뷰 진입 시 부드러운 전환 애니메이션
- 완독 시 축하 이펙트 (파티클)
- 스트릭 갱신 시 불꽃 효과
- 새 영역 해금 시 빛 효과

### Phase 4: AI 튜터 NPC (궁극 목표)

**4-1. 아키텍처**
- NPC 대화 시스템에 AI 백엔드 연결
- **기본**: BYOK (Bring Your Own Key) — 사용자가 API 키 입력
  - localStorage에 전용 키로 저장, Claude API에만 직접 전송
  - 설정에서 "키 삭제" 버튼 제공
- **옵션**: Cloudflare Workers 프록시 (별도 인프라, GitHub Pages 외부 배포)
  - 정적 사이트 제약과 별개의 서버리스 함수로 rate limiting 담당
- 대화 컨텍스트: 현재 읽고 있는 책 + 진행 상태 + 이전 질문

**4-2. 스트리밍 대화 렌더링**
- DialogueNode 렌더러를 `AsyncIterator<string>` 지원으로 확장
- SSE 또는 ReadableStream → React 상태 업데이트 → 캐릭터 단위 RPG 대화 출력
- 기존 정적 대화와 AI 스트리밍 대화 공존 (NPC별 모드 분리)

**4-3. AI NPC 행동**
- 사서 NPC에게 질문하면 현재 읽고 있는 콘텐츠 기반으로 답변
- "이 코드가 이해 안 돼요" → 해당 섹션의 마크다운 컨텍스트와 함께 AI에 전달
- 힌트 모드: 직접 답을 주지 않고 소크라테스식 질문으로 유도
- 학습 수준에 따라 설명 깊이 조절

**4-4. 기술 선택지**
- Claude API (Haiku for 빠른 응답, Sonnet for 심화)
- 프롬프트에 현재 마크다운 콘텐츠 + 사용자 진행 상태 주입

## Scalability: 콘텐츠 파이프라인

develop-study-documents → developer-open-book 이관 프로세스:

```
1. develop-study-documents에서 학습 문서 작성/수정
2. md-to-web 스킬 또는 수동으로 developer-open-book에 복사
   - public/content/{shelf-id}/{slug}.md
3. src/data/books.ts에 엔트리 추가
4. src/data/shelves.ts에 카테고리 등록 (신규 시)
5. npm run deploy → gh-pages 배포
```

**자동화 가능 영역:**
- books.ts 엔트리를 public/content/ 폴더 스캔으로 자동 생성하는 빌드 스크립트
- `vite.config.ts`에 플러그인 추가: 빌드 시 public/content/ 디렉터리 스캔 → books.ts 자동 생성
- 궁극적으로: GitHub Action으로 develop-study-documents push → developer-open-book 자동 배포

**300권+ 성능 고려:**
- 라우트 300개는 HashRouter + lazy()로 문제 없음 (각각 코드 분할)
- books.ts가 커져도 초기 로드에 영향 적음 (메타데이터만, 콘텐츠 별도 fetch)
- search-index.json은 빌드 시 자동 재생성 필요
- 도서관 UI: 방 단위로 나뉘므로 한 화면에 300권 렌더링 없음
